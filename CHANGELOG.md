# Changelog

## 0.8.2 - 2026-07-23

- 优化长时间静默后的自主挂断：连续静默判断不再错误重置累计时长，模型能看到累计静默时间和判断次数；第三次及以后可结合用户可能离开、话题收束、时间、人格和关系，自主先告别再挂断，但不会按固定倒计时强制断线。
- “安静时主动找话题”和“允许模型自然结束通话”不再互相绑定；关闭主动找话题后，长时间无人回应仍可只进行静默结束判断，不会为了填补沉默另起话题。
- 修复通话模型误判当前时段：浏览器发言会附带经过校验的设备本地时间与 IANA 时区，并在本轮提示中优先于可能缓存的陪伴场景时间；旧页面或时间不可用时提示模型保持不确定，不再自行猜测具体时间并催促休息。
- TTS 开始日志新增 `text_source`、`spoken_chars` 和 `visible_chars`，可直接确认是否使用主模型外语直出，无需再通过字符数差异推断。
- 新增“通话外语语音直出”：已接通电话且陪伴 TTS 目标为日语或英语时，主对话模型可在一次生成中同时给出目标语种口语与用户可见正文，直接送入语音合成，避免额外翻译模型造成的等待。
- 外语直出兼容新版 `voice_language` 与旧版 `browser_language` 配置；陪伴插件缺失、配置不可读、中文目标或模型未按语音标签格式输出时，均静默回退原有 TTS 链路，不改变跨插件接口。
- 修复 Docker/Linux 环境点击房间链接后提示“房间页面不存在”：房间服务会优先读取插件目录中的 `web/`，目录布局不一致或插件由 ZIP/特殊加载器运行时自动回退到 Python 包资源。
- 房间服务启动前校验 `index.html`、CSS、JavaScript 和本地图标；资源确实缺失时不再签发必然失败的链接，并在日志中列出检查过的文件目录与包资源位置。
- 发布包继续使用 POSIX 归档路径并显式包含完整 `web/`，Docker 持久卷中残留的不完整旧插件目录可直接使用 `0.8.2` 发布包覆盖更新。

## 0.8.1 - 2026-07-23

- 工作协同从瞬时屏幕陪聊升级为目标闭环：持续维护目标、验收标准、当前步骤、进度、阻碍、下一步与完成证据，用户明确的新目标才会替换当前目标；没有可靠证据时不会误报完成。
- 房间页新增协同执行状态面板，可直接查看当前目标、验收标准、进度、下一步和阻碍；模型内部状态标记会在显示与 TTS 前移除。
- 已有明确目标时，屏幕伙伴上下文发生变化会低频触发一次进度核验；没有新证据保持安静，目标完成后自动停止，避免后台连续消耗模型调用。
- 新增独立 TTS 播放音量：拓展页可配置 `0–100%` 默认值，房间设置可按当前浏览器即时覆盖并记住偏好，同时作用于服务端 TTS 音频和浏览器朗读，不改变影片原声音量、语种或语音合成内容。
- 当前服务端音频会立即响应房间音量滑块；正在进行的浏览器朗读也会尝试原位更新音量，不中断或重播句子，不支持动态更新的浏览器会从下一次发声开始生效。
- 设置面板增加矮屏独立滚动边界，通话主页面保持固定，手机和横屏环境下音量、摄像头与能力状态均可访问。

## 0.8.0 - 2026-07-23

- 新增可选“工作协同”模式：联动 `astrbot_plugin_screen_companion` 3.3.0+ 的工作协同扩展 API，只读取已经存在的脱敏/缩减上下文，不触发新的截图、录屏或视觉模型调用，也不传输原始画面。
- 工作协同入口会按扩展 API 能力动态显示；未安装、未启用或接口不可用时隐藏模式标签，相关指令和工具返回明确的不可用状态，电话与共同观影保持可用。
- 工作协同会将当前应用、场景、窗口和最近观察作为可能过时的资料，优先帮助模型识别目标、进展、阻碍与下一步，并把 `shared_work` 活动同步到屏幕伙伴的活动轨迹。
- 新增“允许模型自然结束通话”开关（默认开启）：模型遵循提示词判断是否自然收束，先输出告别语，仅在语音正常播放完成后结束当前语音连接；语音不可用、播放失败或被用户打断时保持连接，短暂沉默、没听清或没有话题不会单独触发挂断。
- 发布脚本进一步排除运行时数据、测试缓存、覆盖率产物、日志、临时文件和开发环境目录，避免将本机生成文件或旧压缩包带入插件发布包。
- 暂时移除 Windows/Android 独立客户端路线，包括客户端工程、桌面自动发现、专用签票接口和房间页原生客户端适配；浏览器房间与 AstrBot 拓展页保持可用。

- 视频 Range 请求因拖动进度或切换视频而正常断开时不再输出大段 aiohttp 连接重置堆栈。

## 0.5.0 - 2026-07-21

- 浏览器与 AstrBot 语音识别结果会继续立即进入回复流程；回复完成前，左下角识别文字旁显示排除按钮，误识别时可一键取消本次生成并从对话上下文中排除，不增加固定确认等待。
- 修复手机切换前后摄像头时因旧视频轨尚未释放而提示“摄像头占用”；默认明确优先前摄并镜像显示，切换期间禁用按钮，失败时尝试恢复原镜头。
- 群聊触发开房时不再公开一次性链接：aiocqhttp 会将手机邀请私发给发起人，群内只返回确认；QQ 禁止临时私聊时撤销未使用票据并引导用户转到 Bot 私聊，不把链接回退到群内。
- `/一起`、`/一起 看视频` 与自然语言开房工具现在会按需自动启动房间服务和 Cloudflare Quick Tunnel，直接在当前 QQ 会话返回手机邀请链接，不再要求先进入拓展页、手动开启穿透、打开本机浏览器再复制。
- 修复部分手机浏览器虚假声明支持 Web Speech、启动后返回 `not-allowed` 时直接挂断通话；现在优先自动切换 AstrBot STT 的按住说话，未配置服务端 STT 时仍保留文字和摄像头通话。
- 修复拓展页自动打开电脑浏览器后，可复制链接的一次性凭证已被提前消费；房间顶栏新增邀请入口，可随时生成新的完整短期链接。新凭证放入 `/join/` 路径并兼容旧 `?ticket=` 链接，降低 QQ 或浏览器复制时查询参数丢失导致“缺少凭证”的概率。
- 通话房间新增可选摄像头：支持本地镜头预览、前后摄像头切换和低频临时帧缓存；语音、文字与安静时主动回应可结合新鲜画面，关闭镜头或挂断后立即清空且不持久化。
- 多模态对话模型直接接收摄像头与观影帧；对话模型为纯文本时，独立视觉模型只负责受约束的客观转述，最终回复继续由原对话模型结合人格和会话上下文生成。
- 拓展页新增 Cloudflare Quick Tunnel 一键临时穿透：自动生成手机可访问的 HTTPS 房间地址，等待 DNS 生效并持续检查连通性，停止或插件重载时自动回收进程；固定公网地址配置不受影响。
- 对话模型调整为必选，视觉模型保持可选；对话模型支持图片时优先直接理解视频帧，否则才调用独立视觉模型转述。
- B 站解析支持 DASH 高清视频与音频双轨同步播放，实测可获取 1080P；保留合并流作为兼容回退。
- LLM 开房工具要求完整返回带 `/join/` 一次性凭证的手机链接；新房间同时继承当前用户最近的有效对话上下文。
- 拓展页新增常用房间设置，可直接选择通话、视觉、STT 与 TTS Provider，调整语音方式、观影观察节奏、原声协调、上下文和记忆选项；保存后立即同步当前运行实例。
- 新增独立 Token 用量统计：覆盖实时回复、画面问答、自然观影评论、观前背景、临时剧情记忆和共同经历摘要，并通过运行时桥接展示在陪伴面板中。
- 房间凭证改为握手后一次性消费，收紧 WebSocket 同源校验，并在关闭房间与重载插件时统一回收后台任务。
- 优化实时共处与观影判断提示词，使语气更贴合精力、情绪和关系状态，并减少机械解说。
- 房间断线可恢复：连接意外中断后保留房间 180 秒，页面自动重连并恢复观影进度与最近对话；刷新页面同样可恢复。
- 修复长语音消息恰好顶破 WebSocket 上限导致房间被断开的问题；浏览器朗读兜底语言现在跟随识别语言配置。
- 修复 FFmpeg 转换超时后进程残留、流式调用空产出被静默二次计费的问题；跨插件联动统一 5 秒超时，兄弟插件挂起不再冻结房间。
- 降低观影模型调用成本：对话与字幕证据不再在 system prompt 与上下文中重复注入；陪伴场景与 Bot 身份按短 TTL 缓存；高频活动同步按 30 秒节流。
- 页面不可见时暂停自动画面观察，抓帧改为异步编码；Bot 说话链路增加 90 秒看门狗，浏览器识别连续失败改为退避重启。
- 媒体转发复用连接；修复每个房间都在 AstrBot 积累一个新会话的问题。
