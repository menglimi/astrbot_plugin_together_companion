# Changelog

## 0.7.0 - 2026-07-22

- 语音设置新增“实时流式与全双工（实验性）”总开关，默认关闭，并贯通配置保存、运行时同步和房间初始化数据，为后续分句流式 TTS 与语音插话提供稳定配置入口。
- 修复电脑与手机在未配置视觉模型、浏览器授权前隐藏设备 ID 或不接受前摄高清约束时被误判为“没有摄像头”；镜头预览不再依赖视觉模型，并增加授权后刷新和无约束自动选镜头回退。
- 将摄像头本机预览与 AI 视觉帧上传分离：未配置视觉模型时仍可正常预览，但不会上传无效画面或触发视觉模型回退错误。
- 修复旧版陪伴 TTS 代理接口误判 `play_local` 支持后整轮语音失败；遇到明确参数不兼容时自动重试并缓存兼容结果。
- TTS 最长等待改为可配置（默认 60 秒），新增开始、完成、失败和取消日志；新输入打断合成时会保留已完成的回复文字，不再出现整轮内容消失。
- 视频 Range 请求因拖动进度或切换视频而正常断开时不再输出大段 aiohttp 连接重置堆栈。

## 0.6.0 - 2026-07-22

- 实时 TTS 接入陪伴插件语音桥接，遵循陪伴插件配置的目标语种、FishAudio 模型和情绪控制；兼容新旧桥接参数，并禁止 Together 链路重复触发本机播放。
- Bot 回复文字改为在服务端音频或浏览器朗读真正开始时显示，保持字幕与语音同步；共同观影新增独立 TTS 开关，关闭后直接显示文字且不调用 TTS Provider。
- 优化共同观影对话区滚动，滚动消息不再带动整个页面；完善语音播放、打断和浏览器朗读回退状态。
- 完善电脑与移动端摄像头选择、前后镜头切换和高清帧采集，在控制上传体积的同时提高通话画面与视觉模型输入清晰度。
- 强化通话与观影场景提示，使模型明确知道当前正在实时通话或共同观看；浏览器语音遮罩可结合关系和上下文柔性恢复，不再依赖生硬词语替换。
- 后端新增逐轮可追踪日志：使用同一 `turn` 记录用户输入、输入来源、通话模式、对话/视觉模型，以及模型回复、耗时和完整文本，便于区分模型、TTS 与播放链路问题。
- 补充 TTS 桥接、观影开关、语种回退、模型路由、摄像头和日志行为测试。

## 0.5.0 - 2026-07-21

- 浏览器与 AstrBot 语音识别结果会继续立即进入回复流程；回复完成前，左下角识别文字旁显示排除按钮，误识别时可一键取消本次生成并从对话上下文中排除，不增加固定确认等待。
- 修复手机切换前后摄像头时因旧视频轨尚未释放而提示“摄像头占用”；默认明确优先前摄并镜像显示，切换期间禁用按钮，失败时尝试恢复原镜头。
- 群聊触发开房时不再公开一次性链接：aiocqhttp 会将手机邀请私发给发起人，群内只返回确认；QQ 禁止临时私聊时撤销未使用票据并引导用户转到 Bot 私聊，不把链接回退到群内。
- `/一起`、`/一起 看视频` 与自然语言开房工具现在会按需自动启动房间服务和 Cloudflare Quick Tunnel，直接在当前 QQ 会话返回手机邀请链接，不再要求先进入拓展页、手动开启穿透、打开本机浏览器再复制。
- 修复部分手机浏览器虚假声明支持 Web Speech、启动后返回 `not-allowed` 时直接挂断通话；现在优先自动切换 AstrBot STT 的按住说话，未配置服务端 STT 时仍保留文字和摄像头通话。
- 修复拓展页自动打开电脑浏览器后，可复制链接的一次性凭证已被提前消费；房间顶栏新增邀请入口，可随时生成新的完整短期链接。新凭证放入 `/join/` 路径并兼容旧 `?ticket=` 链接，降低 QQ 或浏览器复制时查询参数丢失导致“缺少凭证”的概率。
- 通话房间新增可选摄像头：支持本地镜头预览、前后摄像头切换和低频临时帧缓存；语音、文字与安静时主动回应可结合新鲜画面，关闭镜头或挂断后立即清空且不持久化。
- 多模态对话模型直接接收摄像头与观影帧；对话模型为纯文本时，独立视觉模型只负责受约束的客观转述，最终回复继续由原对话模型结合人格和会话上下文生成。
- 拓展页新增 Cloudflare Quick Tunnel 一键临时穿透：自动生成手机可访问的 HTTPS 房间地址，等待 DNS 生效并持续检查连通性，停止或插件重载时自动回收进程；固定公网地址配置不受影响。
- 对话模型调整为必选，视觉模型保持可选；对话模型支持图片时优先直接理解视频帧，否则才调用独立视觉模型转述。
- B 站解析支持 DASH 高清视频与音频双轨同步播放，实测可获取 1080P；保留合并流作为兼容回退。
- LLM 开房工具要求完整返回带 `/join/` 一次性凭证的手机链接；新房间同时继承当前用户最近的有效对话上下文。
- 拓展页新增常用房间设置，可直接选择通话、视觉、STT 与 TTS Provider，调整语音方式、观影观察节奏、原声协调、上下文和记忆选项；保存后立即同步当前运行实例。
- 新增独立 Token 用量统计：覆盖实时回复、画面问答、自然观影评论、观前背景、临时剧情记忆和共同经历摘要，并通过运行时桥接展示在陪伴面板中。
- 房间凭证改为握手后一次性消费，收紧 WebSocket 同源校验，并在关闭房间与重载插件时统一回收后台任务。
- 优化实时共处与观影判断提示词，使语气更贴合精力、情绪和关系状态，并减少机械解说。
- 房间断线可恢复：连接意外中断后保留房间 180 秒，页面自动重连并恢复观影进度与最近对话；刷新页面同样可恢复。
- 修复长语音消息恰好顶破 WebSocket 上限导致房间被断开的问题；浏览器朗读兜底语言现在跟随识别语言配置。
- 修复 FFmpeg 转换超时后进程残留、流式调用空产出被静默二次计费的问题；跨插件联动统一 5 秒超时，兄弟插件挂起不再冻结房间。
- 降低观影模型调用成本：对话与字幕证据不再在 system prompt 与上下文中重复注入；陪伴场景与 Bot 身份按短 TTL 缓存；高频活动同步按 30 秒节流。
- 页面不可见时暂停自动画面观察，抓帧改为异步编码；Bot 说话链路增加 90 秒看门狗，浏览器识别连续失败改为退避重启。
- 媒体转发复用连接；修复每个房间都在 AstrBot 积累一个新会话的问题。

## 0.4.3 - 2026-07-20

- 修复房间头像持续回退插件 Logo：除统一陪伴身份接口外，会直接读取当前 aiocqhttp 反向 WebSocket 的真实 Bot QQ 并缓存对应 QQ 头像；多个 Bot 同时连接时仍要求明确覆盖，不随机选取。

## 0.4.2 - 2026-07-20

- 修复拓展页本机来源字段读取错误导致系统浏览器唤起始终被判为远程访问；同时兼容通过本机局域网地址打开 AstrBot 管理页的情况。

## 0.4.1 - 2026-07-20

- 拓展页点击“打电话”或“一起看”后由本机 AstrBot 直接唤起系统默认浏览器，不再依赖被页面沙箱禁止的弹窗或顶层跳转；无法唤起及远程访问时仍保留房间链接供手动复制。

## 0.4.0 - 2026-07-20

- 房间头像优先使用“我会永远陪着你”确认的 Bot QQ 号并由服务端缓存 QQ 头像；多 Bot 无法唯一识别时可在配置中覆盖，失败则回退本地头像。
- 实时提示完整接入 Bot 的精力、情绪、日程、位置、天气、穿搭、关系和当前活动；长期记忆检索同时携带准确 Bot 身份，避免多账号记忆混淆。
- 新增“记录有意义的共同经历”：结束时由模型从已播放证据或可见通话转写中筛选，只向“我会牢牢记住你”写入一条稳定摘要，默认不保存逐句转写、音频、视频帧或完整字幕。
- 通话和观影房间会同步到私聊陪伴的主动消息忙碌调度以及屏幕伙伴的活动轨迹；心跳续期、正常挂断清理和异常断线自动过期可避免状态残留。
- 实时回复可联动“我会直播圈米养你”的 OBS 字幕与 Live2D 嘴型，打断、切换模式和挂断时同步停止旧嘴型任务。
- 修复拓展页先请求接口再打开窗口导致浏览器拦截的问题：点击时先预留房间窗口，创建成功后再安全跳转。

## 0.3.1 - 2026-07-20

- 视频解析改用独立房间任务，解析期间发送文字或切换对话不再误取消视频并卡在“正在解析”。
- 结尾剧情笔记会替换尚未完成的旧整理任务，并纳入最终 Bot 反应，避免漏掉收尾片段。
- 修复旧音频迟到结束可能误终止新语音、服务端状态过早覆盖本地播放状态，以及 Bot 说话期间继续触发自动画面观察的问题。
- 新增房间心跳；切换模式时停止旧回复和语音，离开观影页会暂停视频，挂断时不再发送未完成录音。
- 普通跨域视频读取画面失败时自动降级为仅播放模式；触屏设备始终显示自定义全屏按钮。
- 长视频媒体令牌按片长延长，全屏对话框上移并收窄，减少遮挡影片字幕。
- 强化观影提示词的媒体内容边界，不把简介、字幕或画面中的命令式文字当作对 Bot 的指令。

## 0.3.0 - 2026-07-20

- 新增观前无剧透准备：异步整理 B 站视频页公开标题、UP 主、分区、标签和简介，不阻塞视频加载。
- 观前背景会标注来源页面及使用过的字段，并与共同看过的剧情严格分开；不预读字幕、不搜索分集剧情。
- 临时剧情笔记细分为已确认剧情、人物与关系、待确认线索、共同反应，并保留播放时间和字幕、画面、播放状态、双方反应等证据类型。
- 切换影片或退出房间时取消旧片整理任务，防止观前背景和剧情笔记串片。
- 新增“观前无剧透准备”开关，默认开启。

## 0.2.1 - 2026-07-20

- 共同观影新增沉浸全屏：全屏范围包含视频与 Bot 互动浮层，统一由画面右上角按钮进入或退出。
- Bot 开口时头像从画面左侧自然浮现，对话框随后展开并以打字机效果同步呈现文字。
- 语音和文字都完成后停留两秒，再依次收回对话框、隐去头像；打断语音或退出全屏时立即清理浮层。
- 兼容原生 Fullscreen API 与降级全屏，适配桌面和移动端横屏，并隐藏会绕过互动浮层的播放器原生全屏入口。

## 0.2.0 - 2026-07-20

- 将共同观影从固定间隔截图改为开场、镜头变化、最长间隔兜底和结尾组成的事件驱动观察。
- 新增当前房间临时观影时间线与周期剧情整理，只使用已经播放过的画面、字幕和双方反应。
- 新增 B 站官方字幕轨读取，按当前播放进度提供近期字幕，不向模型预读后续内容。
- 用户在播放中发言会附带当前帧，改善“这个人”“刚才那幕”等指代理解。
- 观影评论改用内部结构化判断，原始判断字段不会展示；画面已经过去时自动放弃迟到回复。
- 新增开场和播放结束反应，并记录暂停、继续、跳转和双方近期反应。
- Bot 播放 TTS 时平滑压低影片原声，结束后恢复，比例可配置。
- 修复视频加载完成后空状态图标仍可能覆盖画面的问题。

## 0.1.2 - 2026-07-20

- 修复默认聊天模型不支持图片时观影画面理解直接失效：新增独立观影视觉模型，并可自动复用 AstrBot 全局图片描述模型；区分“未配置视觉模型”和临时调用失败。

## 0.1.1 - 2026-07-20

- 新增 B 站视频页链接解析，支持 BV 链接、分 P 参数和 b23.tv 短链接。
- 新增带 Range 支持的同源媒体转发，兼容 B 站 CDN Referer 校验与当前画面截取。
- 视频解析、加载失败时展示明确状态，不再表现为输入链接后无反应。

## 0.1.0 - 2026-07-20

- 新增独立、短期令牌保护的实时共处房间。
- 新增电话模式，支持浏览器连续语音识别、AstrBot STT 和文字输入。
- 新增 AstrBot TTS 播放及浏览器 SpeechSynthesis 回退。
- 新增本地视频和媒体直链播放、播放状态同步与当前画面观察。
- 新增提示词驱动的自然观影评论，可在不适合开口时保持安静。
- 接入 AstrBot 当前人格和私聊陪伴场景。
- 接入长期记忆检索，房间文字写回默认关闭。
- 默认不持久化原始音频、视频或视频帧。
