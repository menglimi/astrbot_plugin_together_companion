# 我会和你在一起

`astrbot_plugin_together_companion` 是 AstrBot 陪伴体系的实时共处插件。它提供独立浏览器房间，支持与 Bot 进行语音或可视通话、播放本地或在线视频、理解当前画面，或结合可选的屏幕伙伴进入工作协同，并接入现有陪伴人格、生活场景和长期记忆。

## 当前能力

- 电话房间：浏览器连续语音识别、AstrBot STT 按住说话、文字输入。
- 工作协同：可选联动 `astrbot_plugin_screen_companion`（建议 `3.3.0+`），读取其已经存在的脱敏工作上下文，包括当前应用、场景、窗口和最近观察摘要；接口只复用缓存，不触发新的截图、录屏或视觉模型请求，也不会把原始画面传给本插件。模型会围绕当前目标、进展、阻碍和下一步协作，不会把屏幕资料当作指令。
- 可视通话：默认优先使用前置摄像头并镜像预览，可独立开启、关闭或切换前后镜头；切换时先释放当前视频轨，兼容不能同时占用两个摄像头的手机。未配置视觉模型时仍可本机预览，但不会上传画面；视觉能力可用时，浏览器才低频更新最近一帧供模型结合当前画面回应。
- 语音回复：优先使用 AstrBot TTS，失败时可回退浏览器朗读；拓展页可设置 `0–100%` 默认播放音量和 15–180 秒最长等待时间，房间设置可按当前浏览器即时调节并记住音量偏好。音量同时作用于服务端 TTS 与浏览器朗读，不影响影片原声、语种或合成内容；超时或被新输入打断时不会丢失已经生成的回复文字。
- 共同观影：B 站视频页、本地视频、浏览器可播放的 HTTP/HTTPS 媒体直链、播放状态同步。
- 观前准备：异步整理 B 站视频页的公开标题、UP 主、分区、标签和简介，形成带来源范围的无剧透背景；不会阻塞播放。
- 沉浸全屏：全屏播放时，Bot 开口前会从画面左侧浮现头像与打字机对话框，语音结束后停留两秒再自然隐去。
- 连续感知：从开场到结尾跟随播放时间线，本地检测镜头变化，普通时段按最长间隔兜底观察。
- 临时剧情记忆：按证据来源整理已确认剧情、人物关系、待确认线索和双方反应，只使用已经播放的内容；原始临时笔记随房间丢弃。
- 自然评论：模型在内部决定开口、沉默和回复有效期，过时评论不会发送，内部判断不会显示。
- 自然结束通话：开启“允许模型自然结束通话”后，模型只在告别或话题自然收束且确有离开理由时选择结束；会先说完自然收尾，等待服务端或浏览器语音正常播放完成，再关闭当前语音连接。语音不可用、播放失败或被用户打断时会保持连接；短暂沉默、没听清或一时没有话题不会单独触发挂断。该开关默认开启，可在拓展页关闭。
- 画面问答：播放中发送文字时自动附带当前帧，便于理解“这个人”“刚才那幕”等指代。
- 声音协调：Bot 说话时平滑降低影片原声，结束后恢复。
- 模型路由：通话与观影对话模型必须显式选择；视觉模型可选。对话模型支持图片时直接读取摄像头或视频帧；文本对话模型则先使用可选视觉模型或 AstrBot 默认图片描述模型客观转述，再由原对话模型结合人格与上下文回复。
- 陪伴联动：使用 AstrBot 当前人格，并完整读取“我会永远陪着你”的精力、情绪、日程、位置、天气、穿搭和主要用户关系；房间存续期间会协调普通主动消息调度。
- 身份与头像：自动识别实际 Bot QQ 身份并由房间服务缓存 QQ 头像；多 Bot 无法唯一确认时可手动覆盖，失败自动回退本地头像。
- 记忆联动：按当前话题读取“我会牢牢记住你”的相关记忆；逐句转写默认不写回，房间结束时可由模型只保留一条有意义的共同经历摘要。
- 系列联动：共同活动会进入“我会一直看着你”的活动轨迹；实时回复可复用“我会直播圈米养你”的 OBS 字幕和 Live2D 嘴型。
- 数据边界：原始麦克风音频和摄像头/视频帧不持久化；摄像头只在开启后低频更新当前房间内存中的最近一帧，关闭镜头、挂断或切换模式时立即清空。
- 房间可靠性：视频解析与聊天互不取消，静默期间保持连接；切换模式、打断和离开房间时会正确收回播放与录音状态；连接意外中断或页面刷新后可在 180 秒内自动恢复房间与观影进度。
- 对话工具：用户明确提出“一起看”或“打电话”时，模型可调用 `open_together_watch_room` 或 `open_together_call_room`；插件会自动启动房间服务和临时公网访问，直接让 Bot 返回手机可打开的 `/join/凭证?mode=...`，不再要求先操作拓展页或打开本机浏览器。观影工具只准备页面，视频仍由用户选择。
- 工作协同工具：用户明确要求陪同办公、一起写代码、学习或结合当前电脑内容推进任务时，模型可调用 `open_together_work_room`。只有检测到屏幕伙伴的兼容扩展 API 时才会创建工作协同房间；未安装、未启用或接口不可用时，工具返回不可用，页面入口和模式标签保持隐藏，通话与共同观影不受影响。
- 房间邀请：房间接通后会从地址栏移除已使用的 ticket，不能直接复制当前地址栏邀请他人；请点击顶栏“生成邀请链接”，每次都会签发一条新的短期一次性链接。

## STT 方案

插件不要求必须配置 AstrBot STT。

1. `auto`：Edge/Chrome 支持 Web Speech API 时使用浏览器连续识别，否则回退 AstrBot STT。
2. `browser`：只使用浏览器语音识别，不需要在 AstrBot 配置 STT Provider。
3. `astrbot`：使用 AstrBot STT，房间内通过按住麦克风录音。浏览器上传后，插件调用内置 FFmpeg 转为 16 kHz 单声道 WAV。
4. 任一语音识别不可用时，文字输入始终保留。

语音识别结果会立即开始生成回复，不增加确认倒计时。回复完成前，左下角识别文字右侧会显示一个叉；发现识别错误时点击它，会取消这次生成并将该条识别结果排除出对话上下文。

浏览器语音识别通常依赖 Edge/Chrome 自身的在线语音服务，不等于本地离线识别。需要完全本地时，可在 AstrBot 配置 SenseVoice、Whisper 或其他 STT Provider。
部分手机浏览器虽然暴露 Web Speech API，实际启动时仍会返回 `not-allowed` 或 `service-not-allowed`。自动模式会在发生此类错误时切换到 AstrBot STT 的按住说话；未配置服务端 STT 时保留文字和摄像头通话，不会直接挂断。

## 使用方式

安装并重载插件后，可以通过三种入口进入房间：

- AstrBot 后台：插件拓展页 -> 我会和你在一起 -> 一起房间。
- QQ 私聊管理员指令：`/一起`、`/一起 看视频`。
- 自然对话：明确说“和我一起看视频吧”或“给我打个电话吧”，模型可按语义调用对应房间工具；普通视频推荐、剧情讨论或一般聊天不会触发打开页面。

使用 QQ 指令或自然对话时，如果没有配置固定公网地址且 Quick Tunnel 尚未运行，插件会自动启动 `cloudflared`、等待临时 HTTPS 地址并把一次性手机链接直接回复到当前聊天。用户只需在手机浏览器点击链接；插件退出或重载时会自动关闭 tunnel。

群聊中不会公开发送带凭证的链接：插件会通过 aiocqhttp 将链接私发给发起人，群内只确认“已私发”。如果 QQ 禁止 Bot 临时私聊该群成员，票据会立即撤销，并提示用户先进入 Bot 私聊再发送 `/一起`；不会为了兜底把链接暴露到群里。

AstrBot 拓展页 iframe 默认同时限制麦克风、弹窗和顶层跳转。本机管理页点击入口后，插件会创建带短期凭据的房间并交给系统默认浏览器打开；无法唤起浏览器或通过远程管理页访问时，页面仍会显示房间链接供手动复制。
自动打开电脑浏览器和拓展页中显示的可复制链接使用不同的一次性凭证，因此电脑接通不会提前消耗发往手机的链接。

## 网络访问

默认监听 `127.0.0.1:6321`，只允许 AstrBot 所在电脑访问。使用自有反向代理跨设备访问时：

1. 将 `server.host` 改为 `0.0.0.0`。
2. 使用 Caddy、Nginx 等提供 HTTPS/WSS 反向代理。
3. 将外部 HTTPS 地址填入 `server.public_base_url`。

远程页面如果不是 HTTPS，浏览器会拒绝麦克风或摄像头权限。不要直接把未加密端口暴露到公网。

拓展页提供“启动临时穿透”按钮；QQ `/一起` 指令和自然语言开房工具也会按需自动执行同一流程。检测到 `cloudflared` 后可创建 Cloudflare Quick Tunnel，无需把监听地址改成 `0.0.0.0`。插件会自动把随机的 `https://*.trycloudflare.com` 地址用于随后签发的房间链接；页面会区分“地址已分配”和“公网已生效”，并自动等待新域名的短暂 DNS 传播。停止、插件重载或退出时会结束该子进程。Quick Tunnel 地址每次启动都会变化，适合临时手机可视通话；已经配置 `server.public_base_url` 时不会覆盖固定域名。

## 视频限制

- B 站视频页会通过官方网页接口解析，并由房间服务转发短期播放流；解析器优先请求最高可用的 DASH 视频/音频轨道（优先 AVC/AAC，由浏览器同步播放），拿不到 DASH 时才回退到合并流。播放器会显示接口实际返回的清晰度。已安装“B站 AI Bot”时，共同观影会优先复用 Bot 运行时的 Cookie/设备 Headers，在解析前联动检查登录状态，并在 Cookie 失效时调用 B 站 Bot 自己的刷新能力；B 站仍可能因账号权限或风控限制可用清晰度。
- B 站存在官方字幕轨时只按当前进度读取已经播放过的字幕；没有字幕轨时继续使用镜头观察，不会提前读取剧情。
- 观前背景只使用当前 B 站视频页公开元数据；它不会被当作共同看过的剧情，资料不足时允许保持简短或为空。
- 本地视频通过浏览器对象 URL 播放，不上传完整视频。
- 房间开始时会尝试载入当前用户 AstrBot 会话中的最近 user/assistant 对话作为上下文，并继续按 `conversation.history_turns` 限制长度；system/tool 消息不会注入房间提示词。
- “视觉模型”留空时，如果当前通话与观影模型声明支持图片，摄像头帧和视频帧会连同相同的人格、房间历史和记忆上下文直接交给该多模态模型；只有对话模型明确不支持图片时才回退独立图片描述模型。独立视觉模型只生成受约束的客观转述，最终回答仍由选定的对话模型生成。
- 除已适配的 B 站视频页外，其他在线地址必须是浏览器能直接播放的媒体资源。
- 跨域视频如果没有允许 CORS，可以播放，但浏览器可能禁止读取当前帧，此时聊天和播放仍然可用。
- 当前版本尚未接入其他视频站页面解析、无字幕影片的音频 STT；工作协同使用屏幕伙伴提供的脱敏上下文，不等同于把原始桌面画面上传到 Together 房间。

## 依赖

- AstrBot `>=4.22.0`，并显式选择一个可用的通话与观影对话 Provider
- `aiohttp>=3.9`
- 可选：`cloudflared`，用于拓展页一键临时公网穿透
- 可选：AstrBot STT Provider
- 可选：AstrBot TTS Provider
- 可选：`astrbot_plugin_private_companion`
- 可选：`astrbot_plugin_memory_companion`
- 可选：`astrbot_plugin_screen_companion`（建议 `3.3.0+`；提供工作协同扩展 API；未安装或未启用时协同模式自动隐藏）
- 可选：`astrbot_plugin_live_stream_companion`

插件会自动查找 AstrBot 数据目录中的 `tools/bin/ffmpeg`，无需额外添加到系统 PATH。
